# CRM 메시지 자동 작성 & 성과 예측 툴 - 서비스 기획안

## 1. 서비스 개요

### 1.1 서비스 목적
마케터가 앱 푸시 메시지 캠페인을 기획할 때, AI를 활용하여 효과적인 메시지를 빠르게 작성하고 발송 전 성과를 예측하여 의사결정을 지원하는 툴

### 1.2 타겟 사용자
- 이커머스, 리테일 등의 마케터
- 정기적으로 앱 푸시 캠페인을 진행하는 담당자
- 메시지 작성에 시간이 많이 소요되고, A/B 테스트 문구 작성에 어려움을 겪는 사용자

### 1.3 핵심 가치 제안
- **시간 절약**: 수동으로 여러 버전의 메시지를 작성하는 시간을 90% 단축
- **데이터 기반 의사결정**: 발송 전 성과 예측으로 리스크 감소
- **일관된 품질**: AI가 생성한 다양한 메시지 옵션으로 높은 품질 유지

### 1.4 MVP 검증 목표
AI가 생성한 메시지의 품질과 다양성이 실제 마케터의 업무에 유용한지 검증

## 2. 주요 기능

### 2.1 대시보드
**목적**: 과거 캠페인 성과 한눈에 파악 및 새 캠페인 시작

**주요 요소**:
- 최근 캠페인 목록 (제목, 발송일, 발송 시간, 발송 수, 주요 성과 지표)
- 성과 요약 카드 (총 발송 수, 평균 클릭률, 평균 전환율)
- "새 캠페인 만들기" CTA 버튼
- 템플릿 관리 페이지 이동 링크

**사용자 시나리오**:
1. 마케터가 로그인하면 대시보드에서 지난 캠페인 성과 확인
2. 트렌드를 파악하고 새 캠페인 기획 시작

### 2.2 캠페인 작성 페이지 (핵심 기능)
**목적**: 한 페이지에서 캠페인 정보 입력부터 메시지 생성, 성과 예측까지 완료

#### 2.2.1 섹션 1: 캠페인 정보 입력
**입력 필드**:
- 캠페인 제목 (내부 관리용)
- 캠페인 목적 (예: 신상품 프로모션, 재구매 유도, 이벤트 참여 등)
- 타겟 고객 (예: 신규 회원, 휴면 고객, VIP 등)
- 주요 메시지 (전달하고 싶은 핵심 내용)
- 톤앤매너 선택 (친근한, 전문적인, 긴급한 등)
- 포함할 키워드 (선택사항)
- 제외할 표현 (선택사항)

**템플릿 기능**:
- "저장된 템플릿 불러오기" 버튼
- 드롭다운으로 템플릿 선택 시 입력 필드 자동 채움
- 불러온 후 수정 가능

#### 2.2.2 섹션 2: AI 메시지 생성
**동작 방식**:
1. 사용자가 "메시지 생성" 버튼 클릭
2. OpenAI API를 통해 2-5개의 메시지 옵션 생성 (사용자가 개수 선택 가능)
3. 각 메시지 카드 형태로 표시

**사내 메시지 작성 규칙 (항상 적용)**:

AI 메시지 생성 시 다음 규칙이 자동으로 적용됩니다:

1. **메시지 포맷**:
   - 제목
   - (광고) 소제목
   - 내용 1~2줄
   - 수신거부: 마이페이지 > 설정

2. **글자수 제한**:
   - 제목: 최대 한글 기준 20자
   - 소제목: 최대 한글 기준 16자
   - 내용: 1~2줄, 줄당 최대 한글 기준 22자

3. **작성 규칙**:
   - 직관적인 표현 사용 (지나치게 감성적이거나 커머셜한 표현 금지)
   - 제목과 첫째줄은 문장이 연결되지 않고 한 줄씩 끊어져야 함
   - 모든 메시지는 위 포맷과 규칙을 준수하여 생성됨

**메시지 카드 구성**:
- 제목 (글자수 표시)
- (광고) 소제목 (글자수 표시)
- 내용 1줄 (글자수 표시)
- 내용 2줄 (글자수 표시, 선택사항)
- 수신거부: 마이페이지 > 설정 (고정)
- "선택" 버튼
- "재생성" 버튼 (해당 옵션만 다시 생성)
- "수동 편집" 버튼

**추가 기능**:
- 전체 재생성 버튼
- 생성된 메시지 비교 뷰 (나란히 보기)

#### 2.2.3 섹션 3: 성과 예측
**표시 시점**: 사용자가 메시지 중 하나를 선택한 후

**예측 지표**:
- 예상 클릭률 (%)
- 예상 전환율 (%)
- 신뢰도 표시 (예측 정확도)

**시각화**:
- 게이지 차트 또는 프로그레스 바
- 과거 평균 대비 비교 (예: "평균보다 15% 높을 것으로 예상")
- 예측 근거 간단 설명 (예: "유사한 톤의 과거 캠페인 성과 기반")

**데이터 기반**:
- 사용자가 업로드한 과거 성과 데이터 (CSV/엑셀)
- 캠페인 목적, 타겟, 메시지 특성 등을 변수로 활용

#### 2.2.4 섹션 4: 저장 및 내보내기
**기능**:
- "캠페인 저장" 버튼 (대시보드에 추가)
- "템플릿으로 저장" 버튼 (입력 정보를 템플릿으로 저장)
- "메시지 복사" 버튼 (클립보드에 복사)
- "CSV 내보내기" 버튼 (메시지 옵션들을 파일로 저장)

### 2.3 템플릿 관리 페이지
**목적**: 자주 사용하는 캠페인 설정을 템플릿으로 관리

**기능**:
- 저장된 템플릿 목록 (카드 또는 테이블 형태)
- 각 템플릿 정보: 이름, 목적, 타겟, 생성일
- 템플릿 편집, 삭제, 복제 기능
- "새 템플릿 만들기" 버튼

**템플릿 저장 내용**:
- 캠페인 목적
- 타겟 고객
- 주요 메시지 (선택사항)
- 톤앤매너
- 포함/제외 키워드
- 완성된 메시지 문구 (선택사항)

**사용자 시나리오**:
1. 마케터가 매주 비슷한 유형의 캠페인 진행 (예: 주말 특가)
2. 템플릿으로 저장해두면 다음에 빠르게 불러와 일부만 수정하여 사용

### 2.4 설정 페이지
**목적**: 시스템 전체에 적용되는 사내 메시지 작성 규칙 관리

**기능**:
- 메시지 포맷 설정 (고정 텍스트, 순서 등)
- 글자수 제한 설정 (제목, 소제목, 내용별)
- 작성 규칙 텍스트 입력 (AI에게 전달될 지침)
- 금지 표현 목록 관리
- 필수 포함 문구 설정 (예: 수신거부 안내)

**적용 방식**:
- 설정된 규칙은 모든 캠페인의 AI 메시지 생성 시 자동 적용
- 사용자는 캠페인 작성 시 별도로 규칙을 선택하거나 입력할 필요 없음

**기본 설정값** (MVP):
- 포맷: 제목 / (광고) 소제목 / 내용 1~2줄 / 수신거부: 마이페이지 > 설정
- 글자수: 제목 20자, 소제목 16자, 내용 줄당 22자
- 작성 규칙: 직관적 표현, 감성적/커머셜 표현 금지, 제목과 첫째줄 문장 분리

## 3. 사용자 플로우

### 주요 플로우 설명

**플로우 1: 처음 사용자 (템플릿 없음)**
1. 대시보드 → 새 캠페인 만들기
2. 캠페인 정보 수동 입력
3. 메시지 생성 (2-5개 옵션)
4. 메시지 선택 → 성과 예측 확인
5. 만족하면 저장/내보내기

**플로우 2: 반복 사용자 (템플릿 활용)**
1. 대시보드 → 새 캠페인 만들기
2. 템플릿 불러오기 → 일부 수정
3. 메시지 생성
4. 메시지 선택 → 성과 예측 확인
5. 저장/내보내기

**플로우 3: 템플릿 관리**
1. 대시보드 → 템플릿 관리
2. 기존 템플릿 편집 또는 새 템플릿 생성
3. 저장 → 대시보드 복귀

## 5. MVP 범위 및 제외 사항

### 5.1 MVP에 포함
- 앱 푸시 메시지 채널만 지원
- OpenAI API를 통한 메시지 생성
- 사내 메시지 작성 규칙 자동 적용
- CSV/엑셀 업로드를 통한 과거 성과 데이터 수집
- 기본적인 성과 예측 (클릭률, 전환율)
- 템플릿 저장 및 불러오기
- 설정 페이지를 통한 규칙 관리
- 단일 사용자 (로그인 없이 사용 가능)

### 5.2 MVP에서 제외 (향후 고려)
- SMS, 이메일, 카카오톡 등 다른 채널
- 실제 메시지 발송 기능
- 외부 CRM 툴 연동
- 실시간 성과 추적
- 팀 협업 기능 (멀티 유저)
- 고급 분석 및 리포트
- 메시지 A/B 테스트 자동 실행

## 6. 주요 화면 구성

### 6.1 대시보드
- 헤더: 로고, 네비게이션 (대시보드, 새 캠페인, 템플릿 관리, 설정)
- 성과 요약 섹션: 3개 카드 (총 발송 수, 평균 클릭률, 평균 전환율)
- 최근 캠페인 테이블: 제목, 발송일, 발송 시간, 발송 수, 클릭률, 전환율, 액션 버튼
- CTA: "새 캠페인 만들기" 큰 버튼

### 6.2 캠페인 작성 페이지
- 헤더: 뒤로가기, 페이지 제목
- 4개 섹션이 세로로 배치 (스크롤)
  1. 캠페인 정보 입력 (폼)
  2. 메시지 생성 (버튼 → 카드들)
  3. 성과 예측 (차트/지표)
  4. 저장/내보내기 (버튼들)

### 6.3 템플릿 관리 페이지
- 헤더: 뒤로가기, 페이지 제목, "새 템플릿" 버튼
- 템플릿 카드 그리드 또는 리스트
- 각 카드: 템플릿명, 정보 요약, 편집/삭제 버튼

### 6.4 설정 페이지
- 헤더: 뒤로가기, 페이지 제목
- 섹션별 설정 카드:
  1. 메시지 포맷 설정 (제목, 소제목, 내용, 수신거부 문구)
  2. 글자수 제한 설정 (각 필드별 최대 글자수)
  3. 작성 규칙 설정 (AI에게 전달될 지침 텍스트)
  4. 금지/필수 표현 관리 (리스트 형태)
- 저장 버튼
- 기본값으로 초기화 버튼
